 # AI 小说生成系统：阶段概览（简化中文）

本文件以简洁格式总结开发分阶段目标、关键差异与示例命令。原则：先做能验证假设的最小功能，再逐步扩展。

## 概览

| 阶段 | 目标 | 关键新增 | 规模示例 |
|---|---:|---|---|
| 第1阶段（线性） | 生成可读的 3 章短篇 | 设置（世界）→ 结构规划 → 生成（正文） | ~3 章 |
| 第2阶段（质量） | 加入批评—修订循环以提升质量 | Critique + Editor + 滚动摘要 + 检点 | ~10 章 |
| 第3阶段（篇章尺度） | 支持 50 章，防止世界/人物漂移并可恢复中断 | 完整 Arc 管理 + 分层记忆 + sqlite-vec + REST API | ~50 章 |
| 第4阶段（打磨） | 消除 AI 痕迹、平衡节奏与伏笔 | Foreshadowing、Strand Weave、AI Tell 检测 | ~100 章 |
| 第5阶段（产品化） | 为作者提供可用的 Web 工具与运维面板 | React 前端 + 运维仪表板 + 人类审阅队列 | 生产环境 |

---

## 简短指引（第1阶段为主）

- 初始化与工作流示例（在小说目录下执行）：

```bash
python run.py init             # 初始化工作区（在当前目录）
python run.py status           # 显示当前状态
python run.py setup --idea "侦探赛博朋克…"   # 交互式世界/宪章设置
python run.py plan novel       # 生成 novel_plan.json（结构）
python run.py plan arc 1-2     # 生成 arc_01/arc_plan.json（或 arc_01..arc_02 的范围）
python run.py plan chapter 1-3 # 生成章节计划
python run.py generate         # 生成正文（自动从上次进度续跑）
```

- 第1阶段要点：
  - 使用文件作为唯一数据源（constitution.md, world.yaml, arcs/, chapters/）
  - 纯顺序 Python 调度，易于调试与恢复
  - 将复杂功能（Critique、并行、数据库）留到后续阶段

---

## 每个阶段的核心区别（速览）

- 第1阶段：聚焦可用性与可复现的生成流水线（可交互设置、结构规划、章节生成）。
- 第2阶段：加入 Critique + Editor 与短期记忆（滚动摘要），以系统化提升质量并能度量改进。 
- 第3阶段：引入完整 Arc 管理、分层记忆、语义检索与 REST API，支持长篇运行与断点恢复。 
- 第4阶段：专注风格与结构打磨（伏笔追踪、节奏/情感曲线、AI痕迹检测）。
- 第5阶段：面向作者的产品化交付（前端、运维视图、人类审阅工作流）。
